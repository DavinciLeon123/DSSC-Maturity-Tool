# Plan 07-03: Public Landing Page

## Goal

Implement the full "Start pagina" from Figma as the app's public home route at `/`. The page has four sections: hero (title + subtitle + 2 CTA buttons), feature cards, info section, and footer. The current `/_auth/index.tsx` page (the navy "MAMI Compliance Checker" landing) is replaced by a redirect, and the real landing page moves to the root route `/`.

## Context

- Currently `/_auth/index.tsx` serves `/` (because `_auth` layout matches the root). It shows a basic navy page. This becomes a redirect to `/` public landing.
- A new `frontend/src/routes/index.tsx` file must be created for the public `/` route (no auth guard — public).
- Footer (dark navy, with links) appears only on public pages — it is embedded in the landing page component, NOT in `_app.tsx`.
- Figma node for "Start pagina": `81:1468`.
- `routeTree.gen.ts` must be updated to register the new `/` route (per project convention: file is manually updated before `tsc` build; Vite dev server regenerates it on restart).

### Update `frontend/src/routeTree.gen.ts` to register the new `/` route

The project convention is to manually update `routeTree.gen.ts` before `tsc` build (Vite dev server regenerates it on restart, but the committed file must be up to date).

1. **Restart the Vite dev server** (`npm run dev` in the `frontend/` directory) — this regenerates `routeTree.gen.ts` automatically to include the new `index.tsx` route.
2. After the dev server starts and `routeTree.gen.ts` is updated, **verify** that `/` appears as a route in the generated file.
3. If the file does not update automatically, manually add the `IndexRoute` export following the same pattern as existing routes in the file.

**Do not manually edit `routeTree.gen.ts` unless the auto-regeneration fails.**
