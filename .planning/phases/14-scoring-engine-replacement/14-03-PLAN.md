<objective>
Adapt the four report endpoints (`/report`, `/report/data` GET+POST, `/report/pdf`, `/report/mail`) and the admin `/heatmap` endpoint to stop depending on the ZEN/MoSCoW subsystem, add the new `dimension_scores` field to `/report/data` (D-05), enforce the SCOR-04 completion gate on all four report endpoints (D-07), delete `report_generator.py`'s matrix/heatmap/recommendation builders outright (D-01a), remove the degraded-banner mechanism (D-05a), and reduce `/admin/heatmap` to a fixed trivial degraded response (D-01b). Adapt the affected tests to match.

Purpose: Surface the new dimension scores through the report JSON contract's data endpoint and close SCOR-04 on the report surface, while deleting all dead MAMI-matrix code Phase 16 will rebuild from scratch — without touching the frontend or the Jinja2/PDF template (Phase 16's job).
Output: modified `reports.py`, `report_generator.py`, `admin.py`; adapted `test_reports.py`, `test_report_generator.py`, `test_admin.py`.

This plan runs in parallel with Plan 14-02 (Wave 2) — disjoint files. Both leave deps.py's now-unused `get_zen_engine`/`get_mami_config` for Plan 14-04 to delete; the app stays importable at wave end.
</objective>
