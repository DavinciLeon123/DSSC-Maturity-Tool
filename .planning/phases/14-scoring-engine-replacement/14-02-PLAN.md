<objective>
Repurpose the existing `POST /initiatives/{id}/score` endpoint (D-04): drop the ZEN/MoSCoW `FindingRead` response shape and its `get_zen_engine`/`get_mami_config` dependencies, and return the new per-dimension shape computed by the Plan 14-01 service, gated by the completion check (SCOR-04). Add the endpoint's first-ever automated test coverage (`test_scoring.py` did not previously exist).

Purpose: Surface the new equal-weight scores through the endpoint the frontend's `/score` call already hits, and close the SCOR-04 gap that this endpoint currently has (it returns HTTP 200 all-zeros today, not 422).
Output: modified `backend/app/api/v1/scoring.py`; new `backend/tests/api/test_scoring.py`.

This plan runs in parallel with Plan 14-03 (Wave 2) — the two touch disjoint files (this: scoring.py; that: reports.py/report_generator.py/admin.py). Both leave deps.py's now-unused `get_zen_engine`/`get_mami_config` definitions in place for Plan 14-04 to delete; the app stays importable at the end of this wave.
</objective>
