<objective>
Land the new data model that Phases 14/15/16 build on: a new Assessment entity (draft/submitted lifecycle, created lazily on first answer), a read-only v1 archive model mirroring the old answer shape, a reshaped questionnaire_answer keyed by assessment_id with an integer 1-5 score, a schema_version tag on Initiative, nullable participant_type on Initiative/User plus its Pydantic/admin-row fallout, an assessment-first upsert flow, and a score range validator. Adapt reports.py/admin.py/report_generator.py to the new answer shape so the app boots and the suite stays green.

Purpose: Leg C of the sequenced legs (RESEARCH). The Assessment-first schema is load-bearing for Phase 14 (per-assessment scoring), Phase 15 (autosave/retake/history), and Phase 16 (per-assessment report contract) — designing it now avoids a second answer-table migration when Phase 15 lands (D-06). The actual Alembic migration + verification is 13-04; this plan defines the models/schemas so conftest's create_all-built test schema exercises them.
Output: new Assessment/archive models, reshaped answer model + schemas, schema_version + nullable participant_type, adapted downstream modules, a schema-validation test.
</objective>
