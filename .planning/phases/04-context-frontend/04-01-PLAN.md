    - Shows the followup.prompt as a label above the checkboxes

    - Props: `{ question: Question; answer: LocalAnswer | undefined; onAnswerChange: (questionId: string, value: AnswerValue) => void; onFollowupSelectionsChange: (questionId: string, selections: string[]) => void; onFollowupOtherChange: (questionId: string, text: string) => void }`
    - White card with box-shadow, border-radius, padding 1.5rem, marginBottom 1rem
    - Shows question.text in bold navy color
    - Renders AnswerButtonGroup below text
    - Conditionally renders FollowupPanel when:
      a) question.followup exists AND
      b) answer?.answer_value is "YES" or "NOT_THERE_YET"
    - When answer changes to NOT_APPLICABLE: the parent handler must clear followup_selections and followup_other (QuestionCard itself just calls onAnswerChange — the clearing logic is in WizardPage)

    **4. StepPills.tsx** — Category progress indicator:
    - Props: `{ categories: Category[]; currentCategoryIndex: number; completedCategoryIds: Set<string> }`
    - Row of circles (flex, gap 0.5rem, marginBottom 2rem)
    - Each circle: 2rem x 2rem, border-radius 50%, centered text
    - Active category: navy background, white text, shows category number
    - Completed (not active): green background, white checkmark
    - Incomplete (not active): light gray (#E5E7EB) background, gray text, shows number
    - Render `categories.length` pills dynamically (do NOT hardcode 4)

    **5. WizardPage.tsx** — Main wizard wrapper with navigation state:
    - Props: `{ config: QuestionnaireConfig; initiativeId: number; savedAnswers: AnswerRecord[] }`
    - State: `categoryIndex` (useState 0), `topicIndex` (useState 0)
    - Derived: `currentCategory = config.categories[categoryIndex]`, `currentTopic = currentCategory.topics[topicIndex]`
    - Local answers state: `Record<string, LocalAnswer>` — initialized from savedAnswers on mount
    - `completedCategoryIds`: computed from savedAnswers — a category is complete when ALL required questions across ALL its topics have a saved answer_value
    - **Save on navigate**: Before advancing (Next) or going back, save all dirty answers in the current topic via saveAnswer API calls (Promise.all). Use the existing `saveAnswer` from `lib/questionnaire.ts`.
    - **Forward blocking**: "Next" button disabled when current topic has required questions without answers in localAnswers. When on last topic of a category, also check all prior topics in that category for completion before advancing to next category.
    - **Navigation logic**:
      - Next: if not last topic in category, advance topicIndex. If last topic in category and not last category, advance categoryIndex + reset topicIndex to 0. If last topic of last category, save and navigate to `/dashboard`.
      - Back: if topicIndex > 0, decrement. If topicIndex === 0 and categoryIndex > 0, go to previous category's last topic.
    - **Answer change handler**: When answer changes to NOT_APPLICABLE, clear followup_selections to null and followup_other to null in localAnswers for that question. Pass the cleared values directly to localAnswers state update (do NOT read stale state — see Pitfall 4 in RESEARCH.md).

    - Render QuestionCard for each question in currentTopic
    - Render Back button (disabled on first topic of first category) and Next/Finish button at bottom
    - Show "Finish" label on the Next button when on last topic of last category
