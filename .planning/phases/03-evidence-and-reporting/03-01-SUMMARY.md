## Decisions Made
- Wrote Alembic migration manually instead of autogenerating — PostgreSQL lives in Docker and is not reachable from the local dev shell; the manual migration follows the exact same column and index structure Alembic autogenerate would produce
- Rendered EvidenceInput below the RJSF QuestionnaireForm rather than inside it — RJSF renders questions as a single form schema with no per-question injection hook; rendering evidence inputs after the form per dimension is cleaner and avoids RJSF internals
- 12 of 19 questions have has_evidence=true — selected questions where a public URL adds concrete verification value (governance documents, participant registries, machine-readable schemas, service catalogues, SLAs, API specs)
