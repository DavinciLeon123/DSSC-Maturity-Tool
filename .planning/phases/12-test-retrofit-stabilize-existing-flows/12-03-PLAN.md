<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| authenticated user → /admin API | Only ADMIN-role tokens may cross; the `require_admin` dependency is the enforced boundary |
| admin API → Postgres (raw SQL) | `admin.py` runs raw `text()` SQL against native ENUM columns — the reason D-01 mandates real Postgres |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-12-03-PRIV | Elevation of Privilege | `require_admin` on every admin endpoint | high | mitigate | Parametrized regression test asserts a plain USER token gets 403 on every admin endpoint — pins the access-control boundary so a later phase adding an endpoint without `require_admin` fails the suite. |
| T-12-03-CASCADE | Tampering | `_delete_user_cascade` manual FK order | high | mitigate | Regression test asserts zero orphaned child rows in Postgres after delete — catches a broken/re-ordered manual cascade that would otherwise leave dangling data. |
| T-12-03-ADMINDEL | Elevation of Privilege | `delete_user` self/admin-deletion guard | medium | mitigate | Test asserts deleting an ADMIN target returns 403, pinning the guard against removing the last admin. |
| T-12-03-EXPORT | Information Disclosure | `export_dataset` CSV contents | low | accept | Export is admin-only (covered by T-12-03-PRIV) and contains only fields already visible to admins; CSV shape is characterized, not hardened, this phase. |
</threat_model>
