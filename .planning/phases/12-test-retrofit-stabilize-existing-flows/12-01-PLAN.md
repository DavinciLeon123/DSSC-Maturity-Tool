## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| package registry → build/CI | Dev dependencies are fetched from PyPI during `uv add`; a malicious/typosquatted package would execute in the build and CI environment |
| Docker daemon → test process | testcontainers launches a Postgres container the tests connect to; the container is ephemeral and local/CI-only |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-12-SC | Tampering | `uv add` of pytest/testcontainers/faker/etc. | high | mitigate | Blocking-human legitimacy checkpoint (Task 1) before install; pin exact audited versions; verify resolved `uv.lock` matches the RESEARCH.md audit. Never auto-approvable. |
| T-12-01 | Information Disclosure | test Postgres container | low | accept | Container is ephemeral, bound to localhost/CI, seeded only with synthetic faker data (D-03) — no PII, torn down after the session. |
| T-12-02 | Tampering | crypto libraries | low | accept | This plan introduces NO new crypto library; bcrypt/PyJWT remain untouched (V6 characterization-only per RESEARCH.md Security Domain). |
</threat_model>
