## Decisions Made

- **_app.tsx vs _app/_layout.tsx**: Using `_app.tsx` as a flat layout file instead of `_app/_layout.tsx`. The `_layout.tsx` approach creates a second pathless route that resolves to `/`, conflicting with `_auth/index.tsx`. TanStack Router plugin threw a "Conflicting configuration paths" build error.
- **routeTree.gen.ts committed**: Committed to repo because `tsc -b` runs before Vite in the build script, and the generated file must exist at TypeScript compile time.
- **about.tsx added**: Not in the plan's file list, but Sidebar references `/about`. Without this route, clicking "About" would silently fail. Added as Rule 2 (missing critical functionality).

### Auto-fixed Issues

**1. [Rule 1 - Bug] _app/_layout.tsx caused path conflict with _auth/index.tsx**
- **Found during:** Task 2b (vite build verification)

## Issues Encountered

**routeTree.gen.ts not pre-generated**: The Vite plugin generates this file when `vite` runs, but `tsc -b` runs first in the build script. Resolved by running `vite build` first (which generates the file), then the full `npm run build` (tsc + vite build) passes cleanly.

**Docker venv shadowing**: The `docker-compose.override.yml` mounts `./backend:/app` for hot reload, which hid the container's `.venv`. Fixed by placing venv at `/opt/venv` via `UV_PROJECT_ENVIRONMENT` and copying uv's managed Python to the runtime stage. Commit `2476320`.
